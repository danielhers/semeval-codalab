Data

The training, development and test data available here for English, French and German is not identical to the UCCA data available on GitHub. For the purpose of the shared task, the Time (T) category was merged with the Adverbial (D) category. That is, all the instances of T were replaced by D. In addition, all data here is pre-annotated with features calculated by spaCy 2.0.11 and UDPipe 1.2 (using UD v2.2 models).

The data is released in xml, amr, conllu, sdp and txt formats, each in a separate zip file. Within each zip file, there is a separate directory per corpus.

Note that in German, the whole book is annotated and is split into the training/development/test data.

Statistics

Competitions are held in three languages: English, German and French. For English, we use the Wikipedia UCCA corpus (henceforth Wiki) and the UCCA Twenty Thousand Leagues Under the Sea English-French-German parallel corpus (henceforth 20K Leagues), which includes manual UCCA annotation for the entire book on the German side, and UCCA manual annotation for the first five chapters on the French and English sides. The statistics for the corpora are given in the following table.

All datasets were manually annotated and reviewed by a second annotator.

In addition to the in-domain test set, the English part of the 20K Leagues corpus serves as an out-of-domain test set. In German, the train, development, and test sets are taken from the 20K Leagues corpus.

Given the small amount of annotated data available for French, we only provide trial, development and test sets for this setting. We expect systems for French to use semi-supervised approaches, such as cross-lingual learning or structure projection using the parallel corpus, or relying on datasets with related formalisms such as Universal Dependencies. We also release the full unannotated 20K Leagues corpus, tokenized and aligned (automatically) in English and French, in order to facilitate participants to take cross-lingual approaches. The unannotated text is available here.

Validation

We provide a validation script that can be used on system outputs. Its goal is to rule out cases that are inconsistent with the UCCA annotation guidelines. For example, a Scene, defined by the presence of a Process or a State, should include at least one Participant. Given a file or directory of files, they can be validated using the validate.py script: after installing the ucca package in Python (pip install ucca), run:

python -m scripts.validate FILENAMES

Note that system outputs do not need to pass validation.

Data sets are released in XML format, which includes the tokenized text and automatic pre-processing using spaCy and UDPipe (see here), and for the train and development sets, also their gold-standard UCCA annotation.1 An example XML is given below.
